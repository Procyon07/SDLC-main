Git and GitHub

Git is a Version Control system, which allows you to manage source code history, it is a local solution.

GitHub is a hosting and collaboration provider, is also a web based solution.


Repository is the location where code history is stored.

Branch is a "folder" within the repository containing commits.

Commits are different stages of your code within a branch.


Git does not save every single saved version, it tracks changes between versions instead, saving memory.

Commands

$git init

Initializes a local Git repository

$git add.

Adds all the files of the project folder into git.

$git commit -m ""

Commit changes of current code as a version of the code you want to save, every commit must come with a comment, which should be put between the "".

$git log

Shows change logs.

$git status

Shows current status of the project.

$git branch

Shows current branch.

$git master

Returns you to the master branch.

$git reset --hard "commitid"

Resets branch to selected commit.

$git checkout -- .

Reverts all unstaged changes.

$git checkout -b "new branch name"

Creates a new branch, replicating current repository.

$git merge "branch name"

merges a branch into the master branch.

$git branch -D "Branch name"

Deletes a branch.

$git remote add

Establish a connection between local repo and remote repo.

$git remote add origin "github link"

Adds Repository to given github as origin branch.

$git push

Push local branches to remote repository.

$git clone "git link"

Clones entire repositry into local machine.

$git fetch

Gets repository code from github repo to remote tracking branch.

$git merge

Merge code from remote tracking branch into local branch.

$git pull

Combines git fetch and git merge.
